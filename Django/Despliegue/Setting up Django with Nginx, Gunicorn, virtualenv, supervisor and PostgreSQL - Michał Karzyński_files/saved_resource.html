                                                                                            postrational Mod Dhiraj 3 years ago

                                                                                            The upstream directive is only valid in the http context.
                                                                                            Normally the include statement is placed in an http section of your nginx.conf, like so:

                                                                                            http {
                                                                                                (...)
                                                                                                include /etc/nginx/sites-enabled/*;
                                                                                            }

                                                                                            Perhaps you're trying to include your config outside of the http section, which means that you would be trying to use the upstream directive inthe global context, which is not valid.

                                                                                                                                                      postrational Mod John Kevin Basco 3 years ago

                                                                                                                                                      It's not really a file, but rather a UNIX socket. One program can write to a socket (for instance Nginx can write a request) and another program can read from the socket (e.g. Gunicorn can read the request). The communication is bi-directional, so a response travels over the socket in the opposite direction. This is similar to a network connection (network socket), but slightly faster (no need to create and monitor TCP packets for each piece of exchanged information).

                                                                                                                                                              Let's say you have two domains: foo.com and bar.com and you want to host sites for both from one web server. You set the DNS records for both domains to point to the same IP of your server. Based on that it forwards the request to a virtual server, whose server_name matches the requested domain.
